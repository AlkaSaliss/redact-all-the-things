# Security policy

## Reporting a vulnerability

Do not report security vulnerabilities in public GitHub Issues, Discussions,
pull requests, or the Wiki.

Use GitHub's private vulnerability reporting for this repository:

<https://github.com/AlkaSaliss/redact-all-the-things/security/advisories/new>

Include:

- affected component and revision;
- reproduction steps or proof of concept;
- expected security impact;
- any suggested mitigation;
- whether the issue has been disclosed elsewhere.

You should receive an acknowledgement within seven days. Public disclosure
must wait until a fix or coordinated disclosure plan is available.

## Supported versions

The project is pre-release. Only the current `main` branch is supported.

## Sensitive documents

Never attach real sensitive documents, credentials, personal data, or
production secrets to an issue or pull request. Use generated, synthetic test
fixtures for bug reports and regression tests.
