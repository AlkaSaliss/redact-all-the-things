# Architecture decision records

Architecture decision records preserve durable technical decisions and their
trade-offs. Records are append-only history: when a decision changes, add a new
ADR and mark the previous record as superseded.

## Records

- [ADR 0001: Adopt documentation-as-code and GitHub governance](0001-documentation-and-governance.md)

## Creating a record

Copy [the MADR template](template.md), assign the next four-digit number, and
include links to the associated GitHub Issue, OpenSpec change, and pull
request.
